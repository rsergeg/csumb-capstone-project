# CSUMB Certificate-Track Capstone Workspace

This repository is a working space for my CSUMB certificate-track capstone project. It is intended to keep planning, progress tracking, certification work, study logs, evidence, projects, deliverables, and portfolio materials organized in one advisor-friendly location.

This repository is separate from my CSUMB ILP Portfolio website. The website is the public-facing portfolio experience; this repository is the planning, documentation, evidence, and project workspace used to support the capstone.

## Repository Overview

The current structure includes:

- `00 Dashboard.md` - main capstone dashboard with quick links, current focus, hour summary, and evidence checklist.
- `01 Timeline.md` - week-by-week capstone planning timeline.
- `02 CSUMB Milestones.md` - CSUMB milestone tracking for update, artifact, and final presentation readiness.
- `03 Weekly Work Log.md` - weekly work and hour tracking.
- `Certificates/` - certificate-specific planning notes.
- `Certifications/` - detailed certification workspace, currently including AZ-900 notes and supporting folders.
- `CSUMB Updates/` - draft milestone updates and presentation notes.
- `Deliverables/` - capstone deliverable checklists.
- `Evidence/` - screenshot, certificate, and portfolio evidence tracking.
- `Hours Tracking/` - certification and capstone hour logs.
- `Portfolio/` - portfolio index and artifact planning.
- `Projects/` - applied project notes for labs, automation, endpoint management, and final capstone work.
- `Resources/` - reference notes and terminology.
- `Templates/` - reusable templates for logs, labs, certificate modules, and portfolio artifacts.

## Advisor Review Guide

Advisors should start with these files:

1. `00 Dashboard.md`
2. `01 Timeline.md`
3. `02 CSUMB Milestones.md`
4. `03 Weekly Work Log.md`

These files provide the clearest overview of the capstone plan, timeline, milestone readiness, and weekly progress. Supporting evidence and project work can then be reviewed through the linked folders for certifications, projects, evidence, deliverables, and portfolio materials.

## Privacy and Evidence Handling

Screenshots, certificate evidence, transcripts, exam results, and cloud administration materials should be reviewed before being shared with advisors or added to a public portfolio.

Before sharing evidence, check for and redact any sensitive information, including:

- Personal email addresses or usernames
- Azure subscription IDs, tenant IDs, resource IDs, or account details
- Billing, invoice, or payment information
- Exam candidate IDs, private score reports, or transcript identifiers
- Internal system names, local paths, tokens, credentials, or secrets

Raw or private evidence that should not be committed can be kept in a local private folder that is excluded by `.gitignore`.
