
# Cloud Computing — Basic Idea

- Cloud computing = accessing computing resources over the internet
- Resources include:
    - Storage
    - Processing power
    - Databases
    - Software/apps
    - Networking
- Users do **not** need to own physical hardware
- Businesses pay only for what they use (“pay-as-you-go” model)

### Example

- A company gets a sudden increase in website traffic
- Instead of buying more servers:
    - They scale cloud resources up instantly
    - Website stays online and responsive

---

# Microsoft Azure

- Azure is Microsoft’s cloud computing platform
- Provides:
    - Virtual machines
    - Storage
    - Databases
    - Networking tools
    - Pre-built applications
- Azure acts like a digital marketplace for cloud services
- Helps businesses quickly deploy and manage applications

---

# Cloud Service Models

## 1. Infrastructure as a Service (IaaS)

### Definition

- Renting basic computing resources in the cloud

### Includes

- Virtual machines
- Storage
- Networking

### Key Point

- User controls the infrastructure similar to physical hardware

### Example

- Creating virtual Windows or Linux servers in Azure

---

## 2. Platform as a Service (PaaS)

### Definition

- Cloud-based development environment

### Key Point

- Developers focus on building apps
- Cloud provider manages infrastructure

### Benefits

- Faster development
- Less maintenance
- Increased productivity

### Example

- Hosting a web app without managing servers

---

## 3. Software as a Service (SaaS)

### Definition

- Software accessed directly through the internet

### Key Point

- Most user-friendly model
- No installation or maintenance required

### Examples

- Webmail
- Online photo editors
- Microsoft 365

---

# Benefits of Cloud Computing

## Virtualization

### Definition

- Running multiple virtual computers on one physical machine

### Azure Technology

- Uses Hyper-V virtualization

### Benefits

- Better efficiency
- Reduced hardware costs
- Easier resource management

---

## Elasticity

### Definition

- Ability to quickly scale resources up or down

### Benefits

- Handle traffic spikes
- Avoid wasting resources
- Pay only for what is needed

### Example

- Increasing storage during high website traffic

---

## On-Demand Self-Service

### Definition

- Users manage resources themselves through a portal

### Benefits

- Faster deployment
- No waiting for IT staff
- Greater flexibility

---

# History of Cloud Computing

## 1960s

- Time-sharing systems introduced
- Multiple users accessed one computer

## 1970s

- Internet development enabled remote access

## 1990s

- Virtualization technologies improved
- Multiple VMs could run on one server

## 2000s

- Rise of web applications and online services
- Email and file sharing became common

## Modern Day

- Platforms like Azure became widely used
- Businesses moved infrastructure to the cloud

### Goal

- Protect data and applications in the cloud
