
# Cloud Computing Basics

- internet-based computing
- access resources online
- storage
- processing power
- software/services
- no physical server management

## Main Idea

- resources available on demand
- similar to utilities
- use when needed
- pay for usage

---

# Cloud Service Models

## IaaS

- Infrastructure as a Service
- virtual servers/storage/networking
- user manages operating systems/apps
- flexible/customizable setup

### Examples

- virtual machines
- cloud storage
- networking services

---

## PaaS

- Platform as a Service
- prebuilt development environment
- operating systems included
- databases/tools included

### Main Benefit

- faster app development
- less infrastructure management

---

## SaaS

- Software as a Service
- ready-to-use applications
- browser/web-based access
- no installation/setup needed

### Examples

- webmail
- online editors
- cloud apps/services

---

# Virtualization

- multiple virtual machines on one server
- better hardware usage
- lower costs
- improved efficiency

## VM

- Virtual Machine
- software-based computer system
- acts like physical computer

---

# Scalability

- increase/decrease resources easily
- storage scaling
- processing power scaling
- pay only for what is used

## Important Benefit

- avoids over-provisioning
- avoids under-provisioning

---

# Self-Service Cloud

- user-managed resources
- less IT dependency
- cloud dashboards/interfaces
- faster deployment/changes

---

# Advantages of Cloud Computing

## Cost Efficiency

- pay-as-you-go pricing
- less hardware purchases
- lower maintenance costs

## Scalability

- resources adjusted quickly
- supports changing workloads

## Agility

- faster development/deployment
- easier innovation
- rapid business adaptation

## Collaboration

- shared cloud tools
- remote teamwork
- location-independent access

## Security

- cloud security services
- data/application protection
- Azure security features

---

# Cloud Computing Challenges

## Security Concerns

- data privacy worries
- cloud trust concerns
- sensitive data risks

## Vendor Lock-In

- dependence on one provider
- difficult migration/switching
- possible future cost increases

## Internet Dependency

- internet required for access
- outages affect availability
- connectivity very important
