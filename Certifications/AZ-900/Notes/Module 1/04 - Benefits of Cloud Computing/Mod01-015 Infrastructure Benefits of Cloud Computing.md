
## Why Businesses Move to the Cloud

- Traditional on-premises IT is expensive.
- Companies must buy servers, install hardware, pay for electricity, maintenance, and software updates.
- As a business grows, more hardware is needed, which can lead to wasted resources and higher costs.

---

# Cost Benefits

## Resource Virtualization

- Cloud providers create multiple virtual servers on one physical server.
- Gives access to:
    - Virtual CPUs (vCPUs)
    - Memory (RAM)
    - Storage
- Resources are available on demand.

## Multi-Tenancy

- Multiple customers share the same physical infrastructure.
- Each customer remains isolated and secure.
- Helps reduce overall infrastructure costs.

## Pay-As-You-Go Model

- Pay only for resources actually used.
- No need to purchase excess hardware "just in case."
- Reduces wasted spending.

## Cost Optimization Strategies

### Rightsizing

- Choose the correct amount of resources for a workload.
- Avoid paying for unnecessary CPU, memory, or storage.

### Auto Scaling

- Automatically adds resources during high demand.
- Automatically removes resources during low demand.

### Spot Instances

- Use unused cloud resources at a discounted price.
- Very cost-effective.
- Can be interrupted by the cloud provider.

# Scalability Benefits

## What is Scalability?

- Ability to increase or decrease resources based on demand.
- Helps businesses handle changing workloads efficiently.

## Horizontal Scaling

- Add more virtual servers when demand increases.
- Remove servers when demand decreases.
- Improves flexibility and fault tolerance.

## Auto-Scaling Groups

- Automatically adjust resources based on predefined rules or triggers.
- No manual intervention required.

### Scaling Methods

#### Predictive Scaling

- Uses historical data to predict traffic spikes.
- Adds resources before demand increases.
- Example: Online store preparing for Black Friday.

#### Buffer Scaling

- Keeps a small pool of spare resources available.
- Extra resources activate when demand suddenly rises.
- Spare resources are removed when no longer needed.

#### Spot Instance Scaling

- Good for long-running jobs like:
    - Data analysis
    - Scientific simulations
- Saves money if interruptions are acceptable.

# Flexibility Benefits

## Remote Access

- Employees can access applications and data from anywhere with internet access.
- Supports:
    - Remote work
    - Hybrid work
    - Distributed teams

## Business Continuity

- Operations can continue during:
    - Natural disasters
    - Building closures
    - Pandemics
- Reduces downtime and disruption.

---

# Cloud Security Features

## Data Encryption

### At Rest

- Protects data stored on cloud servers.

### In Transit

- Protects data while moving between devices and the cloud.

## Identity and Access Management (IAM)

- Controls who can access resources.
- Users receive only the permissions they need.

## Compliance Standards

Examples:

- GDPR (General Data Protection Regulation)
- HIPAA (Health Insurance Portability and Accountability Act)

Cloud providers often meet strict compliance requirements.

### Shared Responsibility

- Cloud security is shared between:
    - The cloud provider
    - The customer
- Both have responsibilities for keeping systems secure.
