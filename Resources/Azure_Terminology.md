# Azure Terminology

Use this glossary for AZ-900 and future Azure certification notes.

## Core Terms

| Term | Starter Definition | Notes / Example |
| --- | --- | --- |
| IaaS | Infrastructure as a Service provides cloud-based infrastructure such as virtual machines, networking, and storage. | TODO |
| PaaS | Platform as a Service provides managed platforms for building and deploying applications without managing the underlying infrastructure. | TODO |
| SaaS | Software as a Service provides ready-to-use software delivered over the internet. | TODO |
| Availability Zone | A physically separate datacenter location within an Azure region, designed to improve resiliency. | TODO |
| Region | A geographic area where Azure resources and services are available. | TODO |
| Scalability | The ability to increase or decrease resources to meet demand. | TODO |
| Elasticity | The ability for resources to automatically adjust as demand changes. | TODO |
| High Availability | The ability of a system to stay available and reduce downtime during failures or maintenance. | TODO |
| CapEx | Capital expenditure, usually upfront spending on physical infrastructure. | TODO |
| OpEx | Operational expenditure, usually ongoing spending for services or usage. | TODO |
